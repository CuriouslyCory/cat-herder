## Codebase Patterns
- Three.js is installed as a dep (not devDep); @types/three is devDep
- next/dynamic with ssr:false must be used in a 'use client' component (GameLoader.tsx) — Next.js 16+ requires this
- Game modules live under src/game/; never import from src/app/ or src/server/ from within src/game/
- WorkOS auth: withAuth({ ensureSignedIn: true }) in server components; pass only {id, firstName, email} to client
- Config: CONFIG is frozen (Object.freeze), runtimeConfig is mutable deep copy for debug menu
- ECS Entity is a numeric type alias (not a class); query() results are cached and invalidated on add/remove
- SceneManager is the only module that may import directly from 'three' (besides CameraController)
- PhysicsEngine uses downward raycasts for ground detection with 0.05u snap tolerance
- MovementSystem reads from runtimeConfig (not CONFIG) so debug tweaks apply at runtime
