## 2026-04-29 - US-012
- Created `src/game/systems/CollisionSystem.ts` — broad + narrow phase ECS collision detection
- Broad phase: AABB overlap in XZ plane (2D top-down); skips pairs with no layer/mask intersection
- Narrow phase: circle–circle (push apart by overlap+skinWidth, half-each for dual-dynamic or one-sided) and circle–box (closest-point-on-AABB + push-out)
- Trigger pairs tracked with `pairKey()` (smaller entity id first for symmetry); `trigger:enter`/`trigger:exit` emitted via EventBus when pair state changes
- Solid collisions also emit `collision:enter`/`collision:exit` via EventBus
- Layer/mask filter: `(A.layer & B.mask) !== 0 && (B.layer & A.mask) !== 0` — both directions must pass
- Mutates `Transform` in-place — downstream systems (RenderSystem, CameraController) see corrected positions automatically
- pnpm typecheck and pnpm build pass
- **Learnings for future iterations:**
  - CollisionSystem works in XZ plane only (isometric top-down). Y-axis collision is handled exclusively by PhysicsEngine's ground raycast. Do not mix the two.
  - `pairKey(a, b)` always puts the smaller entity first — ensures both `(a,b)` and `(b,a)` produce the same key for `Set` de-duplication.
  - Trigger and solid collision state machines use the same enter/exit pattern: snapshot new contacts, diff against previous frame's set, emit enter for new, emit exit for gone.
  - Circle-inside-box edge case (distSq < epsilon): fall back to pushing along X axis to avoid NaN from normalizing a zero vector.
  - Transform mutation is the correct output; the Game loop runs CollisionSystem before RenderSystem so corrected positions reach the renderer in the same frame.

## 2026-04-29 - US-003
- Created `src/game/engine/EventBus.ts` — typed pub/sub around the GameEvent discriminated union
- `on<T>(type, handler): Unsubscribe` — adds handler, returns removal fn
- `off<T>(type, handler)` — targeted removal (no-op if not registered)
- `emit(event)` — snapshots bucket before iteration to avoid dispatch reentrancy issues
- `clear()` — removes all listeners; intended for module teardown (HMR, route change)
- Internal storage: `Map<string, Set<AnyHandler>>` where `AnyHandler = (event: GameEvent) => void`; boundary cast via `handler as unknown as AnyHandler` to satisfy TS strict contravariance
- **Learnings for future iterations:**
  - TS strict contravariance rejects direct cast from `Handler<T>` to `Handler<UnionType>` with TS2352. Solution: double-cast through `unknown` (`handler as unknown as AnyHandler`).
  - Snapshotting the Set with `[...bucket]` before iterating prevents handlers added during `emit()` from running in the same cycle (safer default).
  - `clear()` must be called in `destroy()` / teardown to prevent listener leaks across React StrictMode double-mounts and HMR cycles.
