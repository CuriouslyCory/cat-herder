## Codebase Patterns
- ESLint flat config (eslint.config.js) must scope typed rules to `files: ["src/**/*.ts"]` — the config file itself isn't in tsconfig and will error if type-checked
- The project uses `typescript-eslint` (unified package v8+) not the older `@typescript-eslint/eslint-plugin` + `@typescript-eslint/parser` split packages
- pnpm is the package manager; pnpm lint, pnpm typecheck, pnpm build are the quality gates
- `_`-prefixed parameters are allowed by ESLint (configured via argsIgnorePattern) but TypeScript's noUnusedLocals does NOT honor `_` prefix for class fields — keep unused fields out
- Array access without noUncheckedIndexedAccess returns T not T|undefined — `!` non-null assertions on `arr[i]` will be flagged as unnecessary
- Use `type` imports (`import type`) for all type-only imports to satisfy isolatedModules
- `async` functions that have no await expressions will be flagged by `require-await` — make them sync when no await is needed
- React JSX files must be `.tsx`; plain `.ts` files cannot contain JSX syntax
- The `import.meta.env` Vite types require `/// <reference types="vite/client" />` in files that use them (or a global vite-env.d.ts)
- main entry is `src/main.tsx` (not main.ts); index.html must reference the correct extension
