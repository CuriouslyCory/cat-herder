## Codebase Patterns
- `next/dynamic` with `ssr: false` must live in a `'use client'` component in Next.js 16+ (Turbopack). Server Components cannot use it directly. Pattern: server page → client loader with dynamic import → actual component.
- The main repo is on `main` (Next.js T3 app). Branch `feat/engine-foundation` is a stale worktree with an old Vite-based game. Working branch is `feat/engine-foundation-v2` from main.
- Three.js lives under `src/game/engine/`. Game source is cleanly separated from `src/app/`, `src/server/`, `src/trpc/`.
- Use `pnpm typecheck && pnpm build` as the quality gate (no lint script yet — added in US-019).
