## 2026-04-29 - US-003
- Created `src/game/engine/EventBus.ts` — typed pub/sub around the GameEvent discriminated union
- `on<T>(type, handler): Unsubscribe` — adds handler, returns removal fn
- `off<T>(type, handler)` — targeted removal (no-op if not registered)
- `emit(event)` — snapshots bucket before iteration to avoid dispatch reentrancy issues
- `clear()` — removes all listeners; intended for module teardown (HMR, route change)
- Internal storage: `Map<string, Set<AnyHandler>>` where `AnyHandler = (event: GameEvent) => void`; boundary cast via `handler as unknown as AnyHandler` to satisfy TS strict contravariance
- **Learnings for future iterations:**
  - TS strict contravariance rejects direct cast from `Handler<T>` to `Handler<UnionType>` with TS2352. Solution: double-cast through `unknown` (`handler as unknown as AnyHandler`).
  - Snapshotting the Set with `[...bucket]` before iterating prevents handlers added during `emit()` from running in the same cycle (safer default).
  - `clear()` must be called in `destroy()` / teardown to prevent listener leaks across React StrictMode double-mounts and HMR cycles.

## 2026-04-29 - US-004
- Created `src/game/engine/SceneManager.ts` — Three.js isolation boundary
- `SceneHandle` is a `symbol` (opaque by nature — callers cannot reach `THREE.Object3D`)
- `MeshConfig` supports box / sphere / cylinder / plane; optional `dims` tuple overrides per-geometry axes
- `camera` is a public settable field so `CameraController` can swap in an `OrthographicCamera` without SceneManager knowing its type
- `resize()` handles both PerspectiveCamera (aspect ratio) and OrthographicCamera (just updateProjectionMatrix — CameraController owns the frustum math)
- `screenToWorld()` normalizes to NDC, casts a ray, returns first intersection as `Vec3 | null`
- `dispose()` disconnects ResizeObserver, walks the mesh map calling `removeMesh()` (disposes geometry + material), then disposes renderer
- **Learnings for future iterations:**
  - `Symbol("SceneHandle")` is the cleanest opaque handle — no class needed, impossible to forge or access internals from outside the module.
  - `renderer.setSize(w, h, false)` — the `false` flag prevents Three.js from setting `canvas.style.width/height` which would fight CSS layout.
  - When `camera` is typed as `THREE.Camera` (the base class), `instanceof` checks are needed before accessing subclass-specific methods like `.aspect` or `.updateProjectionMatrix()` specialized behavior. Both Perspective and Ortho do inherit `updateProjectionMatrix()` from Camera though — only the aspect update is Perspective-specific.
  - The ESLint `no-restricted-imports` rule (US-019) will enforce that only this file and CameraController import from "three". SceneManager's public API exports only our own types (Vec3, MeshConfig, SceneHandle, RaycastHit) — no THREE types leak through the boundary.

- Maintains `Map<Entity, SceneHandle>` locally (not relying solely on `Renderable.sceneHandle`) so stale entity cleanup works even after `world.destroyEntity()` removes component access
