## Codebase Patterns
- ESLint flat config (eslint.config.js) must scope typed rules to `files: ["src/**/*.ts"]` — the config file itself isn't in tsconfig and will error if type-checked
- The project uses `typescript-eslint` (unified package v8+) not the older `@typescript-eslint/eslint-plugin` + `@typescript-eslint/parser` split packages
- pnpm is the package manager; pnpm lint, pnpm typecheck, pnpm build are the quality gates
- vite.config.ts is included in tsconfig.json "include" array
